# Kapruka Magic

This project is a modern web application that provides an interactive interface for the Kapruka e-commerce platform. It leverages the Kapruka Model Context Protocol (MCP) to seamlessly search for products, check delivery availability, create orders, and track them. The application is built with Vite, React, TypeScript, and integrates with AI SDKs for advanced capabilities.

## Features
- **Product Search & Details**: Browse and search the Kapruka product catalog with advanced filtering.
- **Delivery Checking**: Check delivery availability and fees for specific cities and dates.
- **Order Management**: Create guest-checkout orders and track their status in real-time.
- **AI Integration**: Built with support for Google GenAI and LangChain to enable natural language interactions.
- **Modern UI**: Styled with Tailwind CSS and animated using Framer Motion.

## Tech Stack
- **Framework**: React 18 with Vite
- **Language**: TypeScript
- **Styling**: Tailwind CSS, PostCSS
- **Animations**: Framer Motion
- **AI & Integrations**: `@google/genai`, LangChain (`@langchain/core`, `@langchain/google-genai`)
- **API**: Kapruka MCP (Model Context Protocol) via Streamable HTTP (JSON-RPC 2.0)

## Getting Started

### Prerequisites
- Node.js (v18 or higher recommended)
- npm

### Installation

1. Install the dependencies:
   ```bash
   npm install
   ```

### Running the Development Server

Start the development server:
```bash
npm run dev
```
The application will be available at the local address provided by Vite (typically `http://localhost:5173`).

### Building for Production

To create a production build:
```bash
npm run build
```
You can preview the production build using:
```bash
npm run preview
```

## Project Structure Overview
- `src/`: Contains the main application source code (React components, App layout).
  - `src/lib/kapruka-mcp.ts`: The core Kapruka MCP Client wrapper. It handles JSON-RPC 2.0 session management, fetches products, checks deliveries, and tracks orders directly from the Kapruka API.
- `server/`: Contains backend or proxy server configurations if applicable.
